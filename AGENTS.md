# Repository Guidelines

## Project Structure & Module Organization

This is a Jekyll podcast site for Yarukinai.fm, published through GitHub Pages. Episode content lives in `_posts/` as Markdown files named `YYYY-MM-DD-{episode_number}.md`. Shared page templates are in `_layouts/`, reusable HTML fragments are in `_includes/`, and the podcast feed is generated by `feed.xml`. Styles are SCSS under `css/`: `main.scss` imports shared `_variables.scss`, `_mixins.scss`, and component partials in `css/blocks/`. Static assets are in `images/`, with host portraits in `images/actors/`. TypeScript automation scripts are in `scripts/`, and episode templates are in `_templates/`.

## Build, Test, and Development Commands

- `docker-compose up`: starts the recommended local Jekyll server at `http://localhost:4000` with future posts and incremental rebuilds.
- `bundle install`: installs Ruby/Jekyll dependencies.
- `bundle exec jekyll serve --future --incremental`: runs the site locally without Docker.
- `bundle exec jekyll build`: builds the static site into `_site/`.
- `pnpm install`: installs Node dependencies for automation scripts.
- `pnpm create-episode`: generates the next episode post from the template.
- `pnpm update-audio` or `pnpm update-audio 325`: updates SoundCloud audio metadata.
- `pnpm type-check`: runs TypeScript checks without emitting files.
- `pnpm build`: compiles the TypeScript scripts.

## Coding Style & Naming Conventions

Use strict TypeScript for `scripts/`; keep imports and interfaces typed, and prefer existing helpers before adding new parsing logic. Markdown episode files must keep YAML frontmatter fields such as `actor_ids`, `date`, `description`, `layout: article`, and `title`. SCSS should follow the existing modular, BEM-like block structure and reuse variables and mixins from `css/_variables.scss` and `css/_mixins.scss`.

## Testing Guidelines

There is no dedicated test suite. Validate changes with `pnpm type-check` for script edits and `bundle exec jekyll build` for site/content changes. For new or edited episodes, preview locally and confirm frontmatter renders, links work, and audio metadata is populated when available.

## Commit & Pull Request Guidelines

Recent history uses short, imperative commit messages such as `Add EP 324`, `Update EP 323 title`, and dependency/security update summaries. Keep commits scoped to one change. Pull requests should describe the content or code change, link relevant issues when present, and include screenshots for visible layout/style changes.

## Agent-Specific Instructions

Do not overwrite generated podcast metadata without checking the current post and RSS context. When adding hosts, update `_config.yml` and place the matching image in `images/actors/`.
