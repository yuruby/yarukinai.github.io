最新エピソードのショーノートを作成してください。以下の手順に従ってください。

引数: $ARGUMENTS
- リンク集（URL）をグループごとに空行で区切って渡す
- 各グループの先頭行がURLでない場合、それをトピック名として使用
- URLのみの場合、ページ内容からトピック名を推定

## 手順

### 1. 最新エピソードファイルの特定
`_posts/` ディレクトリから最新エピソードファイルを特定（ファイル名末尾の数字が最大のもの）。Readで読み込む。

### 2. 入力リンクの解析
引数を解析してグループに分割する:
- 空行でグループを区切る
- 各グループの先頭行がURLでなければトピック名、URLであればトピック名は後で推定
- インデント（スペース2〜4個またはタブ）があるURLはその直前のURLの子リンクとして扱う

### 3. 各URLのタイトル取得
グループ内の各URLに対してWebFetchツールで取得し、ページの `<title>` タグまたは `<h1>` タグからタイトルを抽出する。
- 取得できない場合はURLそのものを仮タイトルとする
- タイトルはサイト名を除いたコンテンツ名を優先（例: 「記事タイトル - サイト名」→「記事タイトル（サイト名）」形式に整形）
- 過去EPの書き方を参考に括弧内にサイト名・媒体名を添える

### 4. ショーノートの構造化
以下のルールで Markdown リストを生成する:

```
- トピック名
  - [リンクタイトル（サイト名）](URL)
    - [子リンクタイトル（サイト名）](子URL)
```

- トピック名は簡潔な日本語で（過去EPの命名スタイルを参考に）
- リンクが1件のみのグループはトピック名＋その1件だけ
- 複数リンクは関連性の高いものを子にネスト
- ネストは最大3段（トピック→リンク→子リンク）

### 5. エピソードファイルの更新
Editツールで以下を更新する:

**`## 話したこと` セクション**
`- opトーク` の後（または `- TODO: トピックを追加` の行）を、生成したショーノートで置き換える。
`- opトーク` 行は残す。

**`description` フィールド**
frontmatterの `description:` を以下の形式に更新:
`{俳優名1}、{俳優名2}のN人で「トピック1」「トピック2」「トピック3」などについて話しました。`
- トピック名は `## 話したこと` の主bullet（トピック名）から代表的な3〜4個を選ぶ
- 俳優名は `create-episode` コマンドと同じ日本語マッピングを使用:
